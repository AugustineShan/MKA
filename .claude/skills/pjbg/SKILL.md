# /pjbg - 买方评级报告研究结论生成器

`/pjbg` 只产评级报告里「研究结论」这一块文本，四件固定：**标题 → 短期 → 中长期 → 投资建议**，并把标题+研究结论+盈利预测全表注入 docx 模板产出正式评级报告。表头字段（股票评级/目标价/预期收益率）、声明都不碰；投资要点/正文部分模板自带「见附件」不内嵌。盈利预测全表（R08）由本 skill 从 forecast CSV 渲染注入（对齐评级报告模板 R08 体例：ROE 在 EPS 前、营业收入/归母净利润标 (百万)、yoy/毛利率/ROE 不带 +、PE/PB/EV/EBITDA 不带 x；年份窗同前端 overview）；投资建议后半段两年数字也从 forecast 取。研究员名从前端「配置和教程」页填的 `MKA_RESEARCHER_NAME` 读，注入 R04 研究员格（空则留空）。

**这是买方报告，不是卖方。** 买方报告是分析师自己的投资观点和定位判断，不是给客户的催化清单。语料对齐 `C:\Users\Sheld\Desktop\评级报告\markdown\评级报告合集.md`（26 份博时内部报告）。写之前先内化 §6.0 的买方红线。

## 0. 输入分层与防污染红线

四层输入，严格分层，方向/数字绝不串层：

| 层 | 来源 | 用途 | 红线 |
|---|---|---|---|
| **A 判断层**（最高权威） | `公司判断和最新观点.md` + 公司根目录日期最新的 `*核心假设.md`（参考稿 `*核心假设参考.md`、LOAD 沙箱 `*核心假设_load*.md` 都不算；正式稿认 `{公司}-{YYYYMMDD}-核心假设.md` 命名） | 多空方向、投资落点、中长期逻辑的唯一权威 | 方向/落点只能从这层来 |
| **B 分析师近期文档** | PJBG 素材区内分析师本人写的纪要/零散判断（区别于 Alphapai/卖方） | 补充/微调 A | 与 A 冲突以 A 为准 |
| **C 事实层** | `Skills素材包\PJBG评级报告素材区\` 内 Alphapai 预收集、卖方报告、第三方高频 | 只供事实数据填短期段、佐证中长期 | **禁止**提取 stance/方向/落点/估值结论/预测数字 |
| **D 数字层** | `Agent\forecast\derived_metrics_annual.csv`（`src.forecast` 产物） | 投资建议后半段两年预测 + 估值 | **禁止**从 C 层卖方预期抄数字 |

三条不可破：

1. 方向/落点只从 A/B，**绝不**从 C（卖方/Alphapai）提取观点。
2. 投资建议数字只从 D（ModelKing），**绝不**从 C 抄卖方预期。
3. 判断缺失→停下问分析师；素材区空→停下让分析师补；forecast 缺→停下让分析师先跑 forecast。**绝不自填、不出半成品。**

> 注意：表头 `股票评级` 字段（如"强烈买入"）与投资建议落点（如"不提供买点"）是两个维度，评级字段由别的模板步骤填，**本 skill 完全忽略表头字段**，落点只从 A/B 取。

## 1. 解析公司目录

从 `$ARGUMENTS` 定位 `D:\MKA\companies\{公司}`（同 /ka）：

1. 精确匹配 `companies\{参数}`
2. 代码匹配 `companies\*_{代码}`
3. 公司名匹配 `companies\{公司名}_*`
4. 命中多个列候选询问；未命中报错停止。

## 2. 读 A 层（判断）

- Read `companies\{公司}\公司判断和最新观点.md` —— stance/落点主权威。不存在或为空模板 → 转向核心假设.md，仍读不到明确 stance 则停下问分析师。
- Glob `companies\{公司}\*核心假设*.md`，取文件名日期最新一份（排除 `*核心假设参考.md` 和 `*核心假设_load*.md`——后者是 LOAD 沙箱 vintage，非正式判断）。
- 提取三件：① 多空方向（看多/看空/中性）；② 投资落点（买点/不提供买点/底仓/观望/…）；③ 中长期结构性逻辑。
- A 层两文件冲突时，`公司判断和最新观点.md` 压 `核心假设.md`。

## 3. 判断缺失门禁（fallback）

读完 A 层，若**方向**或**落点**任一不明确 → 停下，向分析师提问，不自填：

> `{公司}` 的判断文件里没读到明确方向/落点：这次看多还是看空？目前位置给不给买点（买点 / 不提供买点 / 底仓 / 观望）？

拿到回答再继续。已知失败模式：无 stance 时会坍缩成模板化怀疑论——必须停下来问，不许自行定调。

中长期逻辑若 A/B 不足，可用 C 层事实补，但**不得凭空生成结构性判断**；不确定处宁可少写。

## 4. 读 B/C 层（事实）—— 短期段素材

- Glob `companies\{公司}\Skills素材包\PJBG评级报告素材区\**`。
- **空**（目录不存在或无文件）→ 停下，告诉分析师补素材区，不产出：
  > `{公司}` 的 `Skills素材包\PJBG评级报告素材区\` 是空的。短期段需要 Alphapai 高频 / 卖方 / 第三方事实，请先补素材再跑 /pjbg。
- **非空** → 读取：
  - B 层：素材区内分析师本人写的纪要/零散判断，用来微调 A（冲突以 A 为准）。
  - C 层：Alphapai 预收集、卖方报告、第三方高频，**只提取事实**：最新高频/季度实绩、分区域分品类拆分、同比环比。**绝不**从 C 层提取方向/落点/估值结论/预测数字。C 层的"市场一致预期""卖方共识""Bull/Bear steelman"一律不碰。

## 5. 读 D 层（数字）—— 投资建议后半段

- 检查 `companies\{公司}\Agent\forecast\derived_metrics_annual.csv` 与 `forecast_is.csv` 是否存在。
- **缺** → 停下，让分析师先跑 `py -m src.forecast --ticker {代码}`，不产出。
- **存在** →
  - 读 `forecast_is.csv` 的 `period` 列，取**前两期** = 近两个预测年（如 2026/2027；不固定，跟着 forecast 跑出来的期数走）。
  - 从 `derived_metrics_annual.csv` 取这两期的：`revenue`、`revenue_yoy`、`n_income_attr_p`、`n_income_attr_p_yoy`、`pe`。
  - **BOM 坑**：这两个 CSV 是 UTF-8-sig，首列 `period` 带 BOM。Python 读取必须 `encoding='utf-8-sig'`，否则 `csv.DictReader` 找不到 `period` 列（`KeyError`）。Read 工具读时 `period` 列名前可能有一个 BOM 字符，按 `period` 识别即可。
  - 单位转换（与盈利预测表口径一致，已对青啤/赛维样本逐项验证）：
    - 营业收入(亿) = `revenue` ÷ 100，保留 1 位小数（revenue 单位百万）
    - 收入 yoy = `revenue_yoy` × 100，**四舍五入到整数**
    - 归母净利润(亿) = `n_income_attr_p` ÷ 100，保留 1 位小数
    - 归母 yoy = `n_income_attr_p_yoy` × 100，**四舍五入到整数**
    - PE(倍) = `pe` 四舍五入到整数

## 6. 生成四件（买方风格）

### 6.0 买方 vs 卖方红线（违反必改）

- ❌ 禁止 "建议关注 X、Y、Z" / "关注催化：A/B/C" 这类**催化清单**（卖方给客户的 watching list）。
- ❌ 禁止 "维持买入/增持评级" / 目标价框架（卖方评级动作）。
- ❌ 禁止**数据罗列**：不把素材区事实堆成一串带小数的清单。事实溶进叙述，只留支撑观点的关键数。
- ❌ 禁止第三人称套话 / hedge（"公司未来有望..."）。
- ✅ **买方落点用语**：估值保护 / 空间 X% / 底仓配置 / 不推荐 / 关注意义不大 / 参与交易价值有限 / 低吸配置 / 不满足"赔率能弥补质地" / 双击。是分析师自己的"要不要买、什么位置买、空间多少"。
- ✅ **买方语气**：直接、有观点、敢判断。可用"我们""感兴趣的领导"。敢说"不推荐""关注意义不大""没有投资价值"。

### 6.1 数字精度

- yoy：整数，无小数（+10.65%→+11%，-9.5%→-10%）。
- 营收：1 位小数（27.20→27.2，130.54→130.5）。
- 归母/利润：≥1 亿 1 位小数（3.49→3.5）；<1 亿 2 位小数保精度（0.43 亿）——对齐语料小额利润习惯。
- PE：整数（21.37→21）。
- 正文事实只留支撑观点的关键数，不堆砌。

### 6.2 标题（报告标题，买方 call）

一句带张力的判断句，买方语气，隐含或明示 call（不推荐 / 不提供买点 / 底仓 / 空间 X%）。约 15–25 字，禁纯多头。公式：`[近期边际事实] + 但/虽 + [核心矛盾或 call]`。标题是 C 事实与 A/B 观点的合成。参考语料："持续投流压力大，净利率受制于亚马逊竞争加剧可视度差，短期内没有投资价值"。

### 6.3 短期

- 加粗主旨：一句**叙述性边际方向**（增速收敛 / 企稳 / 承压 / 加速），不带或极少带数。方向忠于 A/B。
- 正文：**一段**短叙述（2–4 句），把关键事实溶进故事，只留支撑趋势的数（精度按 6.1）。**不是事实清单**，不堆分品类/分区域全量数据。
- 落点在边际方向，不是水平值罗列。

### 6.4 中长期

- 加粗主旨：一句结构性 thesis。
- 正文：**以 3 点为中枢**（视公司论点多少可 2–4 点，不硬凑、不凑数），编号 + 加粗引导词，每条一个判断 + 简短支撑。讲生意模式 / 赛道 / 格局 / 质地 / 发展阶段。主体来自 A/B 层判断，C 层只佐证。是分析师的**观点**，不是数据列表。

### 6.5 投资建议

- 数字句（D 层两年，紧凑，精度按 6.1）：`预计 {年1} 年销售收入 {X} 亿，yoy {X}%，归母净利润 {X} 亿，yoy {X}%，对应 PE {X} 倍；{年2} 年销售收入 {X} 亿，yoy {X}%，归母净利润 {X} 亿，yoy {X}%，对应 PE {X} 倍。`
- 落点句（**买方**定位观点，见 6.0 用语）：一句，来自 A/B 层 stance。说清"要不要买 / 什么位置 / 空间或风险"，**不说"建议关注"**。
- 数字句 + 落点句拼成投资建议。

输出格式（加粗用 `__xxx__`，对齐 corpus）：

```
__{标题}__

__短期：{边际方向主旨}__

{短期叙述段落，2-4 句，关键数溶进故事}

__中长期：{结构性 thesis}__

1. **{论点}**：{判断+简短支撑}
2. ...

__投资建议：__

{D 层数字句}。{A/B 买方落点句}
```

## 7. 落盘

- `mkdir -p companies\{公司}\内部报告\评级报告\`
- Write 研究结论到 `companies\{公司}\内部报告\评级报告\{公司名}-{YYYYMMDD}-研究结论.md`（日期取今日）。
- 文件内容就是四件文本（标题 + 短期 + 中长期 + 投资建议），不含"研究结论"标题头（§7.5 注入 docx 时标题头已在模板里）。
- 研究结论是派生产物（可从 A/C/D 重生），不是 `核心假设.md` 那种神圣文件；同日重跑直接覆盖同名，不走归档两步。

## 7.5 注入 docx 模板

研究结论.md 写完后，注入空白模板产出正式评级报告 docx：

```bash
py -m src.pjbg_docx --ticker {代码}
```

- 读刚写的 `研究结论.md`（自动找 `内部报告\评级报告\` 下日期最新一份）+ `Agent\forecast\derived_metrics_annual.csv`，注入空白模板。
- 注入四处（**按节标题/标签文本定位，不硬编码行号**）：
  - **R01 报告标题** ← 标题
  - **R04 研究员** ← `get_researcher_name()`（前端「配置和教程」页 `MKA_RESEARCHER_NAME`）；空则留空不报错
  - **R06 研究结论** ← 短期/中长期/投资建议，加粗随 .md（`__line__`→bold 段、行内 `**x**`→bold 子串、其余 reg），字体/字号从模板现有 run 采样
  - **R08 盈利预测全表** ← 10 指标×年份窗 子表，对齐评级报告模板 R08 体例（数据来自 forecast CSV，年份窗 `app_config.rating_report_year_config()`；行序 ROE 在 EPS 前；营业收入/归母净利润标 (百万)；revenue/归母 千分位无小数、yoy/毛利率/ROE 1dp% 不带 +、EPS 2dp、PE/PB/EV/EBITDA 1dp 不带 x；营收/归母行加粗）
- 输出 `内部报告\评级报告\{公司名}-{YYYYMMDD}-评级报告.docx`（与 .md 同名前缀，日期一致）。
- 其余表头字段（公司名/代码/股价/评级/目标价/预期收益率/是否重大推荐等）**留空**，由分析师/其他步骤填；投资要点、正文部分模板自带「见附件」不内嵌；声明不动。
- 前置：研究结论.md 已生成（§7）、forecast 已跑（`Agent\forecast\` 有 CSV）。缺则报错停止，不出半成品。

**模板可替换**（公司常换模板，这是硬要求）：空白模板默认 `.claude\skills\pjbg\template\评级报告模板.docx`，可用 `PJBG_TEMPLATE_PATH` 环境变量覆盖。换模板只需替换该文件——注入按节标题（报告标题/研究结论/盈利预测(年度)）定位，通常不用改代码。新模板若节标题文本变了会显式报错停止，需适配。从一份填好的报告派生空白模板：`py scripts\pjbg_make_template.py --src "<填好的.docx>" --out "<空白模板.docx>"`（派生时自动清洗可变值、保留标签格含「是否重大推荐」、投资要点/正文内容行写「见附件」）。

## 8. 汇报

汇报时说清楚：读了哪些 A/B/C/D 层材料、forecast 取了哪两年哪些数、是否通过各门禁、停在哪个门禁前（若有）、写到哪个 .md 与 .docx 文件、用的哪个模板路径。停在门禁前时把给分析师的提问/补料提示原样转达。
