# /ka — 核心假设启动器

## 第一动作：解析公司目录

从 `$ARGUMENTS` 解析公司，在 `D:\MKA\companies\` 下定位目录：

1. 精确匹配 `companies\{参数}` → 直接命中。
2. 代码（如 `002946`）→ 匹配 `companies\*_{参数}`。
3. 公司名（如 `新乳业`）→ 匹配 `companies\{参数}_*`。
4. 命中多个 → 列出候选，问用户。
5. 未命中 → 问用户。

> 新乳业是测试精调公司，其他公司暂时不管。

## 第二动作：立刻读取公司判断和最新观点.md

公司目录定位后，**在读取任何其他材料之前**，先读取：

```
companies\{公司}\公司判断和最新观点.md
```

**这不是"又一份待读材料"。** `公司判断和最新观点.md` 是分析师唯一手写的判断源，是本次工作的**定调材料**。它里面写的核心 thesis、关键假设、关注点和口径选择，就是"老板交代的视角"。

后续读 Excel、年报、纪要时，全部都要放在 `公司判断和最新观点.md` 的框架下解读，不要另起炉灶、不要和它的定调打架。

如果该文件不存在 → **报错停止**，告诉用户："缺少公司判断和最新观点.md，请先准备。它是本次工作的定调材料，没有它无法启动 /ka。"

## 第三动作：判断 init / modify 模式

Glob 检查 `companies\{公司}\核心假设*.md`：

- 存在 → **modify 模式**，读取最新一份现有底稿。
- 不存在 → **init 模式**。

**只认公司根目录下的 `核心假设*.md`，不认任何子目录里的同名文件。** 例如 `companies\{公司}\WEBCLAUDE\核心假设部分\` 是 web 端打包的源材料区，里面即便出现 `核心假设*.md` 也不能当作现有底稿——那不是已确认的核心假设产物，只是打包素材。Glob 必须用**非递归**匹配 `companies\{公司}\核心假设*.md`，禁止用 `**/核心假设*.md` 这类递归模式去扫子目录。

## 第四动作：读取最新活跃素材

扫描 `companies\{公司}\active_vore\`，只取**时间最新**的那一个文件。

按文件类型读取优先级（若未来有多个）：

- .md → 先读
- .doc / .docx → 其次
- .xls / .xlsx / .xlsm → 最后

**若读取的是 Excel 且存在 "年度和半年度" sheet，优先读该 sheet。** 外部模型通常把盈利模型放在这里。

## 第五动作：读取最新年报

扫描 `companies\{公司}\annuals\`，取最新一年的 `*_年度报告.md`。

**注意**：年报是参考材料，不要通读。只在后续流程中拿不准时才翻对应附注。

## 第六动作：动态加载最新版核心假设生成修改器 skill

扫描 `D:\MKA\skills\`，找到匹配 `核心假设生成修改器_skill_v*.md` 的文件中**版本号最大**的那一份。

例如同时存在：

- `核心假设生成修改器_skill_v17.md`
- `核心假设生成修改器_skill_v18.md`

则取 v18。

读取该文件，作为本次工作的核心假设生成/修改器指令。

## 第七动作：进入核心假设生成/修改器流程

现在你已经拥有：

- 公司判断和最新观点.md（最先读，定调材料）
- 现有核心假设底稿（modify 模式）
- 最新活跃素材
- 最新年报（备用查证）
- 最新版核心假设生成/修改器 skill

接下来按加载到的那份 skill 执行：

1. 给老板 overview（骨架、历史区间、整体判断）。
2. 按收入 → 毛利 → 费用 → below-OP 与税 → 中期 分段停止，逐段先押再问。
3. 处理待办区和收纳区。
4. 收口：聊透了出 `公司名-YYYYMMDD-核心假设.md`；没聊透出 `公司名-YYYYMMDD-核心假设参考.md`。

**产物必须写到公司根目录 `companies\{公司}\` 下，禁止写进任何子目录**（尤其不能落进 `WEBCLAUDE\核心假设部分\` —— 那是 web 端打包源材料区，写进去会自我污染，下次 /ka 会被第三动作的 glob 误认成底稿）。这是和第三动作「只认根目录」配套的硬约束：输出落根目录，才能保证下一轮 modify 模式认得对。
