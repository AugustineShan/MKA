# ModelKing 工作台 — 首次使用说明

欢迎使用 ModelKing（MKA）。这是一套 A 股财务数据**取数 + 配平校验 + DCF 建模**的本地工作台。本说明帮你从零跑起来。

---

## 一、前置准备（每台电脑只需装一次）

1. **Python 3.11+**
   - 到 <https://www.python.org/downloads/> 下载安装。
   - 安装时务必勾选 ✅ **Add python.exe to PATH** 和 ✅ **py launcher**。
2. **Node.js LTS**
   - 到 <https://nodejs.org/> 下载 LTS 版安装。

装完后，打开命令行（cmd 或 PowerShell）能分别跑出 `py --version` 和 `npm --version`，就算就绪。

---

## 二、配置凭证（重要，先做这步）

用记事本打开本目录下的 **`.env`** 文件，填入你自己的凭证：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| `TUSHARE_TOKEN` | ✅ 必填 | 三表取数用。到 <https://tushare.pro/register> 注册，在个人主页复制 token |
| `GLM_API_KEY` | 可选 | 年报智能核对（reconciler）用。不填则 LLM 自动补全不可用，规则匹配仍可用。<https://open.bigmodel.cn/> |
| `KIMI_API_KEY` | 可选 | GLM 限流时兜底。<https://platform.moonshot.cn/> |

**其它行（网址、超时、模型名、thinking 开关等）保持原样，不要改。** 填完保存关闭记事本。

---

## 三、启动

双击 **`入口.cmd`**。

- **首次运行**：自动安装 Python / 前端依赖（几分钟，走清华镜像），然后启动服务。
- 启动后浏览器会自动打开 <http://127.0.0.1:8765>。
- **停止服务**：关掉那个黑色命令行窗口即可。

之后每次使用，双击 `入口.cmd` 直接启动（已装好的依赖会跳过，秒开）。

> `入口.cmd` 是唯一的入口文件：首次自动装依赖、日常直接启动，不用记两个文件。

---

## 四、关于数据（companies 目录是空的）

`companies\` 目录默认是**空的**——本系统不带任何公司数据。要分析哪家公司，先取数（在命令行本目录下执行）：

```
py -m src.data_fetcher --ticker 002946.SZ --force --verbose
```

取完数再做清洗、校验、建模。完整流程与命令见 `CLAUDE.md` 和 `docs\` 目录。

---

## 五、常见问题

- **8765 端口被占 / 上次没退干净**：`入口.cmd` 会自动杀掉占用 8765 的僵尸进程，无需手动处理。
- **npm install 慢**：首次需下载前端依赖，属正常。
- **Python 报"找不到模块"**：确认装的是 Python 3.11+ 且勾了 py launcher；重跑 `入口.cmd` 会自动补装依赖。
- **启动后页面打不开**：等命令行窗口出现 `Uvicorn running` 字样再刷新浏览器；或手动访问 <http://127.0.0.1:8765>。
- **`py` 命令找不到**：说明安装 Python 时没勾 py launcher，重新安装勾上即可。

---

## 六、这个工作台能干什么

- 一家公司一页，只读展示财务三表 + 核心指标 + DCF 预测结果。
- "重算"按钮调用正式 DCF 流程（`src.forecast`），结果写到 `companies\{公司}\Agent\forecast\`。
- 工作台**不是**新建模入口——建模走 `/brkd → /ka → /comp` 技能链（需要 Claude Code 环境，非必需）。
