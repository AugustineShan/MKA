# 年报萃取器 — Skill 契约 v2

> 把一份给监管看的、线性的、淹没在套话里的年报,重构成一座给买方研究员用的、可检索、信息密度极高的**公司事实档案**。
>
> 它挖的是年报里两类**标准化数据流喂不出的非标深度信息**:附注里的**量化明细**(数字的细节),和管理层讨论里的**质化叙事**(数字背后管理层的说法)。
>
> 一句话定位:**它是研究员的"公司记忆",不是又一份待读的长文档。** 一次萃取,永久可查,供所有下游研究环节反复参考。

---

## 〇、这个 skill 到底在干什么

三大财务报表是**骨架**——高度浓缩,"营业成本"就一个数,"营业收入"就一个数,TuShare 收的就是这套标准字段。但买方研究真正的弹药,藏在年报里两个 TuShare 给不了的地方:

**支柱一 · 量化明细(明细金矿)。** 附注的明细表:成本由什么构成、收入怎么分部、费用花在哪、利润里多少是一次性的、未来有哪些确定负担、风险埋在哪。这是数字的细节。

**支柱二 · 质化叙事(管理层讨论)。** MD&A / 经营情况讨论与分析里,管理层对这一年的 account:为什么这个数动了、怎么看行业与竞争、未来打算干什么、对重大事项的官方说法。这是数字背后管理层的说法。

两者都是标准化数据流喂不出、却恰恰是形成判断之依据的东西。这个 skill 做的事,是把它们**萃取 + 重组**,不是"清洗":

- 年报本身已经是文本,但它是一份**给监管看的文档**——组织逻辑是合规披露顺序,有价值的明细与叙事被埋在大量法律套话、风险提示模板、治理章程里。
- 本 skill 把那些**真正有研究价值的明细和叙事**从套话里捞出来、按研究问题归类、忠实结构化、标好出处与归因,丢掉绝大部分噪音,组织成研究员能秒查的档案。

**"非标"是它的本质,"数据库"是它的使命。** 每家公司的明细科目都不一样、每家管理层的话术也不一样(这正是这些东西无法被标准字段收录的原因);但萃取出来的东西必须**可检索、可引用、可回溯**,而不是又一份要从头读的长文档。

## 一、灵魂:忠实的档案员,不是分析师

贯穿全文的铁律,违反它整个 skill 就废了:

**本 skill 只忠实地萃取、归类、结构化、标注出处与归因。它绝不做任何投资判断、不做任何评价、不做任何推断。**

- 它抽出"非经常性损益里有 2 亿处置收益",但**绝不**说"所以利润质量差"。
- 它抽出"商誉 50 亿、减值测试显示有减值迹象",但**绝不**说"要暴雷"。
- 它抽出"前五大客户占比 68%",但**绝不**说"客户集中度风险高"。

**判断早已前置进了契约设计里,运行时不再判断。** "决定萃取前五大客户、不萃取董监高简历"本身是一种研究相关性的判断——但这种判断已经固化在下文那套**研究问题**里(透明、可审查)。运行时,skill 只在这套问题的框架内忠实记录;判断住在契约层,不住在执行层。这和数据底座"facts 层不掺判断"是同一个原则——**萃取层一旦在运行时开始判断,就污染了事实**,还会和核心假设生成器抢活、给出打架的结论。

两根支柱,同一个原则,只是手法不同:

- **量化明细——靠原样抄录守忠实。** 公司原文怎么说就怎么抄,不改科目名、不强行翻译。
- **质化叙事——靠归因守忠实。** 记录的不是"claim 真不真",而是"**管理层声称了 X**"。不记"毛利率因原材料涨价下降"(把管理层的因果当成事实),记"**管理层称**毛利率下降系原材料涨价 + 产品降价所致"。这条 claim 本身是客观事实——管理层确实这么说了;它对不对,是下游开会的活。保留管理层的原始 framing,不背书、不反驳。

档案员可以做的:**忠实记录、归类、标注(出处/口径/年份/归因)、指出客观事实**(如"该表只披露 2023–2024 两年""信用减值含一笔大额单项计提 X 元""管理层未对 2026 年经营计划作出表述"——这些是事实陈述,不是判断)。档案员不能做的:任何"好/坏/高/低/风险/质量"的评价性结论。

## 二、采集什么 · 量化明细:用研究问题驱动,不用字段清单驱动

**关键设计:采集范围不是固定字段清单,而是一组研究员关心的结构性问题。** 用问题驱动,面对一家从没见过的公司(哪怕银行、保险这种报表结构完全不同),也不会因为"清单里没有"而漏掉——它会带着问题去找对应明细表,哪怕那张表长得和制造业完全不一样。

值得萃取的明细,本质都在回答下面五个问题之一。逐问题列出**典型**明细类型(典型,非穷举——遇到能回答该问题的其他明细,同样萃取):

### 问题一 · 钱怎么来的(收入的结构)
- 分部 / 分产品 / 分地区 / 分渠道的收入、成本、毛利率及各自同比(A 股"主营业务分行业/分产品/分地区"表通常四样齐全,是金矿,直接支撑下游的拆细与 mix 判断)
- 前五大客户集中度(占比、是否关联方)
- 主要产品产销量、产能、在建产能与投放节奏

### 问题二 · 成本怎么花的(成本与费用的结构)
- 营业成本构成(按性质:原材料/人工/制造费用/能源/运输;或按产品)
- 销售 / 管理 / 研发费用逐项明细(职工薪酬/折旧摊销/市场推广/平台佣金/差旅等)
- 研发投入明细:总投入、费用化与资本化金额及比例
- 前五大供应商集中度

### 问题三 · 赚的钱真不真(利润质量)
- 非经常性损益逐项明细(每项名称和金额)
- 政府补助明细(与收益相关 / 与资产相关、是否经常性)
- 投资收益构成(联营合营 / 处置 / 理财)
- 信用减值 / 资产减值明细(计提了什么、是否有大额单项计提)

### 问题四 · 未来的负担藏在哪(前瞻性义务)
- 折旧摊销政策(各类资产年限、残值率、折旧方法)
- 股权激励 / 股份支付摊销计划(已授予未摊销金额、未来各年摊销额)
- 有息负债明细与利率(短期/长期借款、债券的金额和利率——下游"财务费用由 BS 倒算"的关键弹药)
- 租赁负债(使用权资产、租赁负债)
- 资本开支计划 / 在建工程

### 问题五 · 风险埋在哪(表外与或有)
- 税收优惠明细(各主体税率、优惠依据、**有效期截止日**——下游税率判断的核心)
- 商誉明细及减值测试(哪次收购形成、是否有减值迹象)
- 对外担保、未决诉讼、或有负债
- 应收账款账龄与坏账计提政策
- 关联交易(金额、定价)

### A 股特有、易漏但关键的几项(归入相应问题)
- 会计政策 / 会计估计变更(改折旧年限、改坏账比例等——盈余管理的直接证据,客观记录变更内容,不评价动机)
- 主要会计数据的重述 / 口径说明(准则切换年份的可比性说明)
- 分季度财务数据表(看 Q4 是否有异常)

> 范围边界:本支柱萃取的是"明细金矿",不是把整本年报结构化。纯合规内容(治理章程、风险提示模板、董监高简历、限售股解禁、股东增减持等)**不萃取**——它们不回答上面任何一个研究问题。判断标准始终是:**这块信息能不能回答上面五个研究问题之一?** 能就萃取,纯套话就丢。

## 三、采集什么 · 质化叙事:管理层讨论的 digest

管理层讨论是全年报里信息密度和套话密度都最高的一段。本支柱**不是把它整段搬过来,而是抽取管理层的 assertions、带归因列成清单**,丢掉纯套话。和量化明细一样用问题驱动,平行的六个问题:

### 问题一 · 管理层如何解释今年的经营结果(归因)
数字背后的 why——这是最值钱的一类。它把"成本结构表里毛利率动了"和"管理层说为什么动"接起来。每条带归因、保留管理层引用的具体(哪个产品/地区/数字)。

### 问题二 · 管理层怎么看行业 / 竞争 / 需求
管理层对行业景气、供需、竞争格局的判断与表述。

### 问题三 · 管理层的经营计划与战略重心(前瞻)
未来打算干什么、聚焦哪块、产能与投入方向。A 股展望普遍模糊,但方向性表述同样萃取;管理层没给展望就如实记"未披露"。

### 问题四 · 管理层对重大事项的官方说法
并购 / 处置 / 重组 / 重大股权变动等,管理层自己的解释口径。

### 问题五 · 管理层自陈的核心竞争力 / 护城河
公司怎么描述自己的壁垒(技术/渠道/品牌/成本/牌照等)。

### 问题六 · 其他有经营意义的点
不落在上面五问、但管理层主动披露且具研究价值的经营性信息(例如某项业务的运营指标、某个一次性事件的来龙去脉、某项政策对公司的具体影响)。这是兜底口,防止有价值的非标叙事因"不在五问里"而漏掉。仍只忠实记录、带归因,不评价其重要性。

> **纪律(质化支柱的命脉):** 每条都挂"管理层称 / 公司将…归因于 / 管理层展望…";尽量保留管理层引用的具体数与对象;纯套话("在董事会坚强领导下""坚持高质量发展""稳中求进")全丢。本支柱本质仍是**抽取**(把 assertions 列成带归因的清单),不是自由概括——一旦开始自由概括,就从档案员滑向了分析师。

## 四、每块萃取必带的元数据(可信、可回溯的前提)

半结构化输出的核心纪律:**每一块萃取出来的东西,除内容本身外,必须带元数据**。缺了它们,下游就无法信任和正确使用这些非标数据。

**量化明细带三元数据:**

1. **出处**——来自年报哪一节、哪张表(可回溯到原文)。
2. **口径**——含两个面向,都要记:
   - **原始科目命名**:公司**自己怎么命名**这个科目,原样保留。一家叫"直接材料"、另一家叫"原材料及外购件",这个差异本身有信息量,不能抹平成"标准名"。
   - **计量基础**:合并 / 母公司、含税 / 不含税、调整前 / 重述后。下游拿了母公司的数去填合并的位置,是会**静默出错**的——这个面向比命名更要命,必须标。
3. **年份覆盖**——这张表实际给了哪几年(很多明细只给当年和上年)。

**质化叙事带两项:出处 + 归因。** 出处(MD&A 哪一节)让人能复查;归因("管理层称")是这一支柱可信的根基——它标明了这是一条 claim 而非事实。

这些元数据是"档案可信"的支柱:出处让人能复查、口径让信息不失真、年份覆盖让人知道时间边界、归因让人分清事实与说法。

## 五、自检:标记可疑的萃取,但绝不修复

原料是 `report_downloader` 出来的 markdown,而年报 PDF → markdown 这一步**有损**:数字会串位、表格会断行。对一个要被核心假设生成器拿去倒算财务费用的档案来说,一个串了位的错值比一个缺失值更危险——缺失会被"完整性说明"捕获,错值会被静默消费。

所以萃取后跑一组**轻量内部一致性校验,只标记、不修复**:

- 营业成本构成各项加总 ≈ 营业成本总额
- 分部 / 分产品收入加总 ≈ 营业收入
- 非经常性损益逐项加总 ≈ 非经损益合计

对不上就如实标:"本表各项加总 = X,与披露总额 Y 不符"。这是**客观事实陈述,不是判断**,完全落在档案员本分内,还能在第一时间逮住萃取错误。

**与三大表的边界:** 若萃取出的明细加总与三大表(TuShare)对不上,萃取器同样只忠实记两边数字("明细加总 = X,三大表 = Y"),**配平是 `annual_report_reconciler` 的活,不是萃取器的活**。萃取器只负责把事实摆清楚。

## 六、输出形态:双表示、按研究问题归档

输出走**人读 + 机读双表示**:

- **人读视图(markdown)**:研究员秒查用,按研究问题归档(不按年报章节顺序),带着问题能直接定位。
- **机读 sidecar(JSON/YAML,或直接进 SQLite)**:核心假设生成器是个 agent,它要程序化地抓"有息负债明细与利率"去倒算财务费用——markdown 表格机器解析易错,所以同一份档案必须有一份结构化副本供机器消费,贴合"LLM 翻译、Python 执行"的底座原则。

**schema 原则:信封固定、payload 弹性。** 明细内容本质非标(硬套字段会在第一家不规则公司上崩),但**信封可以固定**:元数据(出处/口径/年份/归因)、问题归类、表的行列结构,这层定 schema;变的只是 payload 里的具体字段。这既不背叛"非标"本质,又给机器消费一个稳定抓手。

**归档粒度:一份年报 = 一份档案文件**(文件头标 `[年报年度]`)。"跨年看分部 mix 演变"这类需求,由下游或一个独立 view 去缝合多份年度档案——萃取器只对**单份报告**负责,不处理跨年口径重述的对齐(那是另一层的事)。

## 七、在管线中的位置与边界

**输入**:`report_downloader` 已下载的年报 markdown(复用已有的公告下载管道)。

**输出**:公司事实档案(人读 markdown + 机读 sidecar),落进 **facts 层**(客观、机器生成、可复用)。一次萃取、永久可查。

**时序**:在核心假设生成器**之前**。生成器开会时不再翻年报 PDF,而是查阅这座档案。

**消费者**(它是被反复参考的资产):
- **核心假设生成器**——最大消费者,且主要吃机读 sidecar。开会查成本构成、税收优惠、非经损益、有息负债,以及管理层对各数字的归因,全读这座档案,不碰原始 PDF。(生成器里原"定向阅读年报附注"的描述,应改写为"查阅公司事实档案"。)
- **写研报**——引用客户集中度、产能规划、mix 数据、管理层战略表述。
- **复盘**——看历史分部 mix 演变、管理层历年展望的兑现度。

### 和 `annual_report_reconciler` 的边界(必须分清,别合并)

两者原料都是年报,但**目标和产出完全不同**:

| | annual_report_reconciler | 本 skill(年报萃取器) |
|---|---|---|
| 目标 | **补三大表配平的洞** | **建三大表没有的明细库 + 叙事库** |
| 对象 | 三大表里 TuShare 缺失/对不平的**少数字段** | 整座附注**明细金矿** + 管理层**叙事** |
| 触发 | 年度硬校验失败时 | 主动预处理,每家公司一次 |
| 产出 | 单个数字的 approved override(进 clean 宽表) | 半结构化事实档案(进 facts 层) |
| 比喻 | 修补地基的洞 | 在地基上建一座可检索的资料库 |

一个补洞,一个建库。不重叠,不合并。

## 八、一句话总纲

**把一份年报,萃取重组成一座"按研究问题归档、忠实保留原始口径与管理层归因、可检索可回溯"的非标事实档案——既挖附注里的量化明细金矿,也 digest 管理层叙事——人读 markdown + 机读 sidecar 双表示,沉淀进 facts 层,供所有下游研究环节反复参考。它只忠实记录、绝不判断;判断早已前置进研究问题的设计里,运行时只摆事实、不下结论。它是研究员的公司记忆,不是又一份待读的文档。**
