# 年度更新器 - Skill v1

你把一家公司的旧 `核心假设.md` 升级到最新年报这一版。产出一份新的带日期 `公司名-YYYYMMDD-核心假设.md`，旧稿原样留存。

执行前必须加载：

```text
skills/核心纪律_skill_v*.md
skills/核心假设源语言_skill_v*.md
```

`annual-update` 完整继承核心纪律 A1-A7，编辑同一套核心假设源语言 B。它不是 `/ka`，也不是 `/adj incremental`：它是“数据新了 N 年，把旧稿滚过去”的特化操作。

## 0. 位置与边界

正常由 `/annual-update` 启动。外层启动器已经先跑：

```bash
python -m src.init <ticker>
py -m src.defaults_gen --ticker <ticker>
py -m src.annual_update_fetcher --ticker <ticker> --history-end <H> --forecast-md "<旧稿>"
```

本 skill 只消费公司工作台里的：

- 旧 `核心假设.md`
- `Agent/data.db`
- `Agent/defaults.yaml`
- 年报 Markdown
- `annual_update_fetcher` 输出 JSON
- 复盘分诊单，若存在
- `公司判断和最新观点.md`

不联网，不重新拉数据，不编译 yaml1，不重建判断引擎。

## 1. 四条本地铁律

共享纪律 A 是上位真源。本 skill 额外遵守：

1. 旧稿只读，绝不覆写。永远另存带日期新稿。
2. 滚稿阶段绝不拉数据。缺数据就停，指回外层 `/annual-update` 或 `/init`。
3. 绝不静默捏造。拿不到的实际值只有两条路：声明式估算并标 `估算·待校准`，或精确待补旗。
4. 不重造轮子。重定未来复用核心纪律 A 和核心假设源语言 B；结构性补丁转 `/adj incremental`；编译复用 `/comp`；分诊复用复盘 skill。

## 2. 第 1 步：盘点与守门

自动、只读，不改任何文件。

1. 读最新旧稿，提取：
   - H：旧稿历史末年。
   - 显式期、衰减期、永续点。
   - 收入骨架、毛利参数化、费用、below-OP、税率、少数股东、terminal。
2. 读数据实际到达年份 A。
3. 计算 N = A - H。若 N <= 0，报告无需更新并停。
4. 守门：`(H, A]` 任何一年缺 clean_annual 或年报 Markdown，停止并精确报告缺口。
5. 建总账：旧稿所有需要被处理的线、历史、stash、旋钮、时间轴四数都进入清单。
6. 识别按需扩展字段：旧稿有历史序列但 fetcher 默认 24 条未覆盖的行，查 `src/field_registry.yaml` 或 `数据格式参考.md`，传给 fetcher。

总账是完整性的地基：后面每项要么填，要么旗，要么平移，要么转 `/adj incremental`。

## 3. 第 2 步：自动滚标准线

用 fetcher JSON 的 `lines` 把 `(H, A]` 实际值 append 到旧稿各段：

- 收入 headline：直接搬 `revenue`。
- 费用率/税率/比率类：ratio 写成百分比显示。
- below-OP 绝对值：符号原样搬，clean_annual 已是对利润贡献口径。
- 其他财务费用：优先 `Agent/financial_expense.yaml`；缺则年报附注 fallback，仍取不到挂旗。
- 少数股东、归母净利率等历史观察：可作为实际比率搬运，不是预测派生。

同步动作：

- 时间轴四数 +N 平移。
- `knobs` horizon 前移 N。
- values 前移；新增显式期末年留给第 4 步重定。
- 每填一项，在总账划掉一项。

注意：这里计算历史实际费用率/税率，是搬运事实，不违反核心纪律 A3/A7；禁止的是手算未来派生预测序列。

## 4. 第 3 步：估算拿不到的实际

非标业务线原子，如销量、吨价、分线收入，常常年报拿不到。按口径阶梯走：

1. 有新外部模型或清楚披露：按来源直接入历史，标明来源。
2. 只有年报粗分部收入：按旧稿口径对齐，分线收入可落；销量/吨价挂待补。
3. 都没有：使用声明式估算。

声明式估算三法：

- 预测真实化：用旧稿对该年预测作为拆分钥匙，按比例缩放全部线去对齐真实总额。
- 份额延续：用上一真实年份额拆当年真实子总额。
- 量价拆解：估出线收入后，carry 较稳的一个因子，另一个倒出。

铁律：

- 估算锚在真实总额。
- 缩放全部线，不用最后一条残差兜底。
- 永远标 `估算·待校准`。
- 估出的实际不能拿去复盘它自己来源的那条预测。

估算值进入历史段时，状态必须是 `estimated·待校准`，不能冒充年报披露或 clean_annual 事实。

## 5. 第 4 步：重定未来

这一步是人机交互，不全自动。

按核心假设源语言 B 的过表顺序：

```text
收入 -> 毛利/成本 -> 费用 -> below-OP 与税 -> 中期/terminal
```

每块只在“该重看”的行停：

- 复盘分诊标红的旋钮。
- 第 3 步挂旗待填的项。
- 年报新 segment 或旧 segment 消失。
- 新补那一年带来的预测窗口变化。
- 最高权重材料或 thesis 有明确变化。

干净的块可以明确写“无新信息，平移”，并在总账划掉。

重定未来遵守核心纪律：

- A4：先押再问，拍板才落盘。
- A5：参数化先于数值。
- A6：正文与 `knobs` 同源回声。
- A7：未来派生序列交引擎。

若旧稿、真实值、thesis、偏离诊断或估算来源之间冲突，按 A2 与 B7 写“来源与裁决”：候选A、候选B、采用、为什么、未采用方去处。未采用方不能静默消失。

结构性变化，例如新增/删除业务线、改变参数化族、改变 horizon/fade 长度，若超出年度滚轴，举旗转 `/adj incremental`，不静默吸收。

## 6. 第 5 步：收口

自动检查：

- 总账每项是否已划掉。
- 新稿是否包含旧稿每条线、每段历史、每块 stash；只能增，不能无声减。
- 新材料中旧骨架没有的 segment 是否已举旗。
- 每个填不上的实际是否“填或旗”，没有第三条路。
- 正文旋钮与 `knobs` 块是否同源。
- 时间轴四数是否一致。

然后跑 `/comp`，生成新 yaml1 和 forecast。旧稿 + 旧 yaml1 原样留存。

## 7. 收口报告

报告一页：

- 滚了几年：H -> A。
- 自动填的标准线。
- 按需扩展取到的字段。
- 估算项：每条 + 方法 + `待校准`。
- 挂旗待补项。
- 重定项：旧值 -> 新值。
- 平移未动项。
- 结构变化：新增/消失业务线，是否转 `/adj incremental`。
- thesis 兑现度。
- 新 DCF vs 平推 vs 旧稿。
- 新稿路径和旧稿留存路径。

## 8. 常见陷阱

| 陷阱 | 为什么错 | 正确做法 |
|---|---|---|
| 覆写旧稿 | 毁掉复盘基准 | 另存带日期新稿 |
| 数据不齐时自己去拉 | 越界且不可预测 | 停，报告缺口，指回 /init |
| 拿不到就编实际 | 污染历史 | 声明式估算或挂旗 |
| 用最后一条线兜残差 | 遮蔽口径差 | 按比例缩放全部线 |
| 估算实际拿去复盘来源预测 | 自己给自己打分 | 标暂不可复盘，待真数校准 |
| 新 segment 静默丢掉 | 完整性失败 | 举旗转 /adj incremental |
| 改正文忘同步 knobs | fidelity 失败 | 同步重吐 `knobs` |
| 重定未来乱序 | 下游返工上游 | 按 B 的过表顺序 |
