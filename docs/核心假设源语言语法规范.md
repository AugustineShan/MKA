# 核心假设源语言语法规范

本文定义 MKA official markdown 的轻量语法。它不是新的运行时格式，不替代 `skills/核心假设源语言_skill_v1.md` 的 B 系列 runbook；它只把 `/ka`、`/load`、`/brkd`、Alphapai reference 和 `/comp` 共同依赖的 markdown 形状固定下来。

`核心假设.md` 是判断源头，`yaml1` 是派生缓存，Semantic IR 是 `/comp` 的翻译账本，不是新的判断源。

## 1. 三类 markdown

### 候选稿

适用状态：

```text
draft / reference / model-extracted / factpack/reference
```

候选稿只供 `/ka` 裁决，不可直接 `/comp`。候选稿必须包含：

```markdown
## 待 /ka 裁决清单
| 事项 | 候选值/方向 | 证据 | 分歧/缺口 | 建议处理 |
|---|---|---|---|---|
```

这张表是晋升议程，不是正式判断。旧候选稿缺表时，`/ka` 在 overview 补成议程并标为 reference 完整性缺口。

### Official 稿

适用状态：

```text
official
```

official 稿位于公司根目录，是 `/comp` 唯一可作为 official forecast 输入的源文。每个正式计算块使用统一块头：

```markdown
### {名称} [上挂: {科目}; compiler: {family}; status: official]
```

块内标准字段：

```markdown
- 业务含义:
- 参数化:
  - 旋钮:
  - 派生:
- 历史原子:
- 预测判断:
- 三件套:
  - 谁定:
  - 为什么:
  - 来源:
- 来源与裁决:
- 风险/收纳:
```

### Official 裁决回执

official 稿推荐包含：

```markdown
## reference 裁决回执
| 来源 | 事项 | 处理 | official 去处 | 理由 |
|---|---|---|---|---|
```

它记录候选事项如何变成 official：采纳、收纳、缺口或丢弃。没有候选稿时可以省略，但如果本轮读过 reference/draft/model-extracted/factpack，推荐写入。

## 2. B 类信息去向

B 类不是垃圾桶，必须区分三种去处：

- `history`: 某个 leaf 自己的历史原子，必须挂在同一个 leaf。
- `stash`: 不驱动计算但有复盘价值的事实、口径、风险或副拆分。
- `display`: 告诉前端这些 B 类信息应放到主表、副拆分表、Reference 页或技术页。

禁止把 B 类收纳项塞进 A 类 note 来假装保全。

## 3. 末尾 knobs

official 稿末尾必须有 fenced `knobs` 块。`knobs` 是正文旋钮的同源回声，不是 YAML1，不写 yaml1 path，不做会计路径映射。

````markdown
```knobs
horizon: [YYYY1, YYYY2, YYYY3]
knobs:
  - anchor: "#业务线"
    family: growth
    unit: pct
    values: [1, 2, 3]
    status: official
```
````

完整语法以 `docs/knobs块契约.md` 为准。

## 4. 受控词表

### status

```text
draft / reference / model-extracted / factpack/reference / official / estimated·待校准
```

### decision

```text
adopted / stashed / gap / rejected
```

中文显示：

```text
采纳 / 收纳 / 缺口 / 丢弃
```

### source_layer

```text
同权重判断材料 / BRKD / LOAD / Alphapai / init / 年报查证 / 分析师确认 / annual-update / adj
```

### unit

```text
pct / ratio / abs_mn
```

### family

`family` 以 `skills/核心假设源语言_skill_v1.md` §B4 为唯一真源。本文只引用，不扩展。任何上游产物不得自创 family。

### audit_status

```text
audit_clean / reference_only / failed
```

### forecast_status

```text
not_run / skipped_missing_data / ran_ok / failed_after_audit_clean / pending_comp_step
```

## 5. 不做的事

- 不新增运行时 schema。
- 不强制生成 JSON IR 文件。
- 不改变 `yaml1` schema。
- 不改变 `src.forecast`、`calc.py`、`yaml1_cleaner.py` 行为。
- 不让候选稿靠改名、复制到根目录或补 `knobs` 直接晋升。
