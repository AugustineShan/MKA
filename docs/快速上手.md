# 快速上手（Agent / 新人首次冷启动主线）

> 这是一条**端到端主线**：用一家公司从零跑到 DCF。先把全局心智模型建立起来，再按需深入各技能文档。
> 配套真源：`docs/技能简要分类.md`（路由）、`docs/MKA规则导航图.md`（规则索引）、`docs/数据流水线.md`（管线）、`docs/ARCHITECTURE.md`（架构）。

## 0. 心智模型：四站管线

```text
TuShare/cninfo  →  /init   →  /brkd /load  →  /ka  →  /comp  →  Agent/forecast/
   原始数据         历史事实    候选理解       正式判断   机器覆盖层   DCF 结果
```

三条铁律（优先于一切，全文见 `CLAUDE.md`）：

1. **核心假设.md 永不原地覆盖**——改稿先归档旧稿到 `KAhistory/`，再写今日新稿，文件名日期必须是今日。
2. **通用性高于一切**——不把任何一家的形状焊进代码/模板/校验。
3. **TuShare 数据缺口用 reconciler 去年报补**——`target_gt_calc` 默认走 `annual_report_reconciler.py`，不问"要不要人工"。

判断源头只有 `核心假设.md`；`yaml1`、`forecast_params.yaml`、`Agent/forecast/` 全是派生产物，**手改派生产物 = 白干**。

人工筛选是第一门禁：`markdown存储区/` 和各种 cache 不是天然证据入口。只有被人放进同权重判断材料、BRKD/LOAD/Alphapai 完成产物、KA 目录顶层 markdown 或 `/init` 事实索引的内容，才进入本轮 `/ka` 裁决。

KA 目录 `Skills素材包/KA（ALPHAPAI拆出来的东西放在这里）/` 是明确给 `/ka` 的人工入口；顶层所有 `*.md` 都会被读。命名为 `核心假设参考*.md` 的按候选稿裁决，其他 markdown 按信息指引读取。

入口窄，收纳宽：人工筛选门只限制读取范围；已进入本轮的材料里，有复盘价值但暂不入模的信息宁可进收纳区/stash，不得因为减负乱扔。

**主导方向**：`/brkd`、`/load` 先忠实抓业务拆分+详细历史+业务数据（即便不参与计算、只进收纳区），预测旋钮是次要 draft 候选；`/ka` 拿这些产物和分析师裁决预测，同时忠实收集与预测有关的历史防丢数。预测是末，业务结构与历史是本。

## 1. 首选样本：新乳业 002946

项目自带冻结 fixture：`tests/fixtures/company_002946/`（`data.db` + `defaults.yaml` + `yaml1`，确定性 forecast 输入）。真实目录在 `companies/新乳业_002946/`。首次跑通建议用真实目录走全流程，用 fixture 跑 `pytest tests/test_forecast_pipeline.py` 验证引擎闭环。

## 2. 端到端走一遍

### Step 1 — 拉取历史事实：`/init`

```bash
py -m src.data_fetcher --ticker 002946.SZ --force --verbose   # 阶段①：TuShare → raw_tushare/meta
py -m src.clean --ticker 002946.SZ --verbose                   # 阶段②：宽表 + 配平校验 → clean_annual/clean_quarterly
```

或直接 `/init 002946`，编排器自动跑拉取→clean→年报下载→reconcile→plug。

**成功判据**：
- `Agent/data.db` 的 `clean_annual` 通过硬校验（IS 1.1-1.6 / BS 2.1-4.3 / CF 5.1-5.5，残差 < 1 百万元）。
- `Agent/core_metrics_overview.md` 生成；`Agent/defaults.yaml` 生成（唯一 YAML2）。
- 字段覆盖：income=86×期数、balancesheet=150×期数、cashflow=89×期数。

**失败处理**：年度硬校验出现 `target_gt_calc` → 默认强触发 reconciler 去年报补数（铁律 3），不静默放行。`target_lt_calc` 多为 clean.py 分类 bug，不走 reconciler。

### Step 2 — 候选理解：`/brkd` 和/或 `/load`

- `/brkd 新乳业`：读 `Skills素材包/BRKD业务理解器（研报和纪要放在这里）/` 的研报纪要 → 产出 `Agent业务讨论.md`（draft，不拍板）。
- `/load 新乳业`：读 `Skills素材包/LOAD外部EXCEL模型理解器（一次最多一个）/` 的唯一 Excel → 产出 KA 参考稿区 `核心假设参考load_{YYYYMMDD}.md`（load-vintage，不回退到 KA）。

**主导方向**：brkd/load 的首要任务是**忠实抓业务拆分+详细历史+业务数据（即便不参与计算、只进收纳区）**，预测旋钮是次要的 draft 候选。这些拆分与历史是 `/ka` 裁决的弹药，丢一份就少一份。
**边界**：两者只交候选理解，不写正式 `核心假设.md`，不写 yaml1，不跑 DCF。brkd/load/alphapai 参考稿建议落 `Skills素材包/KA（ALPHAPAI拆出来的东西放在这里）/`，命名 `核心假设参考{来源}_{YYYYMMDD}.md`，并包含 `## 待 /ka 裁决清单`。

### Step 3 — 正式裁决：`/ka`

```
/ka 新乳业
```

把同权重判断材料 + BRKD + LOAD/reference + KA 目录信息指引 + `/init` 事实 + 分析师 thesis 收口成正式核心假设。`/ka` 读取 KA 目录顶层全部 `*.md`，其中 `核心假设参考*.md` 按 reference 候选处理，其他 markdown 按信息指引处理。

**主导方向**：拿 brkd/load/alphapai 产物和分析师裁决预测，同时忠实收集与预测有关的历史防丢数——预测会变，历史是锚。
**门禁（必须过）**：时间轴四数、接缝总账、骨架门、数值门。
**reference 晋升**：`/ka` 逐项处理 `待 /ka 裁决清单`，裁成采纳、收纳、缺口或丢弃后，才写 official。
**产物**：公司根目录 `新乳业-{YYYYMMDD}-核心假设.md`，抬头 `状态: official`，末尾 official `knobs` 块。
**边界**：不做旧稿 modify（小改走 `/adj` 或 `/frontend-edit`）；默认只裁利润表 + 业务层盈利模型；不读原始 Excel/PDF。

> ⚠️ 改稿遵守铁律 1：先 `py scripts/ka_archive.py "<旧稿>"` 归档，再 Write 今日新稿。

### Step 4 — 编译 + DCF：`/comp`

```
/comp 新乳业
```

或直接：

```bash
py -m src.forecast --ticker 002946.SZ
```

**年份门禁**：若 `clean_annual` 最新实际年已覆盖核心假设预测起点，`/comp` 立即停，提示先 `/annual-update`。

**产物**（都在 `Agent/`，不在公司顶层）：
- `Agent/yaml1_新乳业_{YYYYMMDD}.yaml` — compiler 输出（人的判断覆盖层）
- `Agent/.modelking/forecast_params.yaml` + `yaml1_clean_report.json` — 中间产物
- `Agent/forecast/` — **唯一正式 DCF 输出**（每次重算先清空再生成）：`forecast_is.csv` / `forecast_bs.csv` / `forecast_cf.csv` / `dcf_summary.csv` / `derived_metrics_annual.csv` 等

**成功判据**：
- `yaml1_fidelity_check` 通过（忠实度初步核对，抓数组长度/路径/抄错/符号反）。
- compiler audit 六段完整：A 类覆盖、B 类保全、路径待核、语义待核、主动覆盖回读、Forecast 状态。
- cleaner 折叠通过（family 词表、margin 互斥、unit_factor 结构化）。
- calc BS/CF 残差为浮点误差级；`backtest` 通过。

**验证引擎闭环**（不依赖真实数据，用 fixture）：

```bash
py -m pytest tests/test_forecast_pipeline.py -v
```

## 3. 已有正式稿后的安全修改

| 场景 | 入口 | 边界 |
|---|---|---|
| 只拨已有 knobs 的少量数值 | `/adj quick` | 不加宽白名单，不改结构/family/horizon/fade |
| 读新增材料后系统性小改 | `/adj incremental` | 推翻骨架转 `/ka` 重建 |
| 前端沙盘试算已拍板，落盘 | `/frontend-edit` | 不读新材料，映射不到 path 立即停 |
| 新年报出，旧预测变历史 | `/annual-update` | 旧稿只读另存，clean 失败即停 |
| 重资产 DA/CAPEX 排程 | `/da` | 轻资产别用 |

## 5. 读懂产物：family × 行可编辑性

核心假设展示页每条业务线渲染成 收入行/同比行/销量行/driver 行，哪些可编辑由 `revenue_family` 决定。权威矩阵见 `docs/yaml1算法模板契约.md` §「family × 前端行可编辑性矩阵」。一句话：**每个 family 恰好露一个真输入为可编辑行，其余派生只读**。
