# 公司文件夹快速导航

本文件给后续 AI 和维护者用:进入 `companies/{公司名}_{代码}/` 后,先按这张地图找材料。公司文件夹是基本面工作台,不是程序运行目录的堆场;人工判断、投研证据、打包输出、建模 Agent 产物必须分层。

## 一句话原则

- 公司根目录是分析师工作台入口,只放高频人工工作件和一级材料分区。
- `Agent/` 是建模 Agent 的运行时文件夹,不是"外部模型"文件夹,也不是人工材料仓库。
- `active_vore/` 是活跃收集/外部模型材料入口,不能搬。
- `WEBCLAUDE/` 是给网页 Claude 上传用的打包输出,可重建,不是源材料首选。
- `公告/`、`研报/`、`纪要/`、`收集/`、`重要文件/` 是投研证据层;不要把程序中间产物塞进去。

## 标准结构

```text
companies/{公司名}_{代码}/
|-- 公司判断和最新观点.md
|-- {公司名}-{YYYYMMDD}-核心假设.md
|-- active_vore/
|-- WEBCLAUDE/
|-- 公告/
|   |-- 年报/
|   |-- 季报/
|   `-- 临时公告/
|-- 研报/
|-- 纪要/
|-- 收集/
|-- 重要文件/
`-- Agent/
    |-- data.db
    |-- defaults.yaml
    |-- financial_expense.yaml
    |-- yaml1*.yaml
    |-- forecast/
    |-- recon/
    |-- Logs/
    |-- OfficialBreakdowns/
    `-- .modelking/
```

不是每家公司都会立刻拥有全部目录,但新建公司工作台时应向这个结构收敛。

## 根目录

| 路径 | 角色 | 查找规则 |
| --- | --- | --- |
| `公司判断和最新观点.md` | 分析师当前观点浓缩 | 做核心假设、modify、年度更新时优先读 |
| `{公司名}-{YYYYMMDD}-核心假设.md` | 人类可读的盈利模型底稿 | compiler 的主要源文;只写 Markdown,不写 YAML |
| 其他根目录散文件 | 例外 | 新文件默认不要放根目录;除非它是高频人工工作件 |

根目录不要放 `data.db`、`defaults.yaml`、`yaml1*.yaml`、`forecast/`、`recon/`、`.modelking/`。这些都属于 `Agent/`。

## 投研材料区

| 路径 | 放什么 | 注意 |
| --- | --- | --- |
| `active_vore/` | 活跃外部模型、Excel、临时建模材料、正在被研究的高频材料 | 关键目录,不要移动或改名 |
| `公告/年报/` | 年报 PDF/Markdown | 年度核对、财务费用附注、历史事实查证优先从这里找 |
| `公告/季报/` | 季报、半年报 PDF/Markdown | 季度事实查证、最新经营变化 |
| `公告/临时公告/` | 临时公告、交易所公告 | 重大事项、股权激励、分红、重组等事件 |
| `研报/` | 券商/外部研究报告 | 预测假设、行业口径、可比公司、观点来源 |
| `纪要/` | 调研纪要、会议纪要、电话会记录 | 管理层口径、经营边际变化 |
| `收集/` | 萃取、摘录、网页资料、补充材料 | 作为中间证据仓,但重要结论应沉淀到核心假设或公司判断 |
| `重要文件/` | 分析师明确认为必须保留的关键文件 | 不要被自动清理流程覆盖 |

## WEBCLAUDE

`WEBCLAUDE/` 是网页端 Claude 上传包目录,由打包脚本生成或刷新。

常见子目录:

- `WEBCLAUDE/核心假设部分/`: 给生成/修改核心假设用。
- `WEBCLAUDE/yaml1编译部分/`: 给编译 yaml1 用。

规则:

- 可以清空重建,不要把它当唯一原件。
- 不要从 `WEBCLAUDE/` 反向覆盖公司源材料。
- 如果源材料和 `WEBCLAUDE/` 冲突,以源材料目录和根目录工作件为准。

## Agent

`Agent/` 是建模 Agent 文件夹,承载机器产物、数据库、编译缓存和正式预测输出。

| 路径 | 角色 | 维护者 |
| --- | --- | --- |
| `Agent/data.db` | 清洗后的财务数据库 | `init`/`clean` 等程序 |
| `Agent/defaults.yaml` | YAML2/defaults,机器平推底座和目标命名空间 | `defaults_gen.py` |
| `Agent/financial_expense.yaml` | 财务费用附注分析档案 | `financial_expense_analyzer.py` |
| `Agent/yaml1*.yaml` | 分析师判断编译后的 YAML1 | compiler 或 `/comp` 链路 |
| `Agent/forecast/` | 正式预测三表和 DCF 输出 | `forecast.py`;重算会清空重写 |
| `Agent/recon/` | 年报核对、override、evidence | reconciler / subagent bridge |
| `Agent/Logs/` | 运行日志、年度更新偏离诊断等 | 程序写入 |
| `Agent/OfficialBreakdowns/` | 官方口径拆分抽取/结构化材料 | 程序或研究链路写入 |
| `Agent/.modelking/` | 内部编译缓存和运行快照 | 程序专用,不要手工维护 |

关键边界:

- `Agent/.modelking/forecast_params.yaml` 才是 `calc.py` 的正式输入。
- `Agent/forecast/` 是正式输出目录,不要把人工底稿放进去。
- `Agent/defaults.yaml` 是 fallback,不是判断上限;分析师拍板的判断应进 `yaml1*.yaml`。
- `Agent/` 下的文件可以供 AI 核对数据状态和下游契约,但生成核心假设时不要把它当投研源材料首选。

## 按任务找目录

| 任务 | 先看 | 再看 |
| --- | --- | --- |
| 生成核心假设 | `公司判断和最新观点.md`, `active_vore/`, `公告/年报/`, `研报/`, `纪要/` | `收集/`, `重要文件/`, 必要时 `公告/季报/` |
| 修改核心假设 | 现有 `*-核心假设.md`, `公司判断和最新观点.md` | 本次新增材料所在目录 |
| 编译 yaml1 | `*-核心假设.md`, `Agent/defaults.yaml`, `docs/yaml1算法模板契约.md`, `docs/数据格式参考.md` | `Agent/yaml1*.yaml` 旧版作 diff 参考 |
| 年度更新 | 旧 `*-核心假设.md`, `Agent/data.db`, `Agent/financial_expense.yaml`, `公告/年报/` | `Agent/Logs/` 中的偏离诊断 |
| 跑预测/DCF | `Agent/defaults.yaml`, `Agent/yaml1*.yaml` | 输出看 `Agent/forecast/` 和 `Agent/.modelking/` |
| 查年报核对问题 | `Agent/recon/`, `公告/年报/`, `Agent/data.db` | `Agent/Logs/` |
| 给网页 Claude 打包 | 源材料目录和根目录工作件 | 输出到 `WEBCLAUDE/` |
| 找人工私有材料 | `active_vore/`, `重要文件/`, `收集/` | `研报/`, `纪要/` |

## 禁止事项

- 不要把 `Agent/` 改名为 `模型/`;它表达的是"建模 Agent",不是外部模型仓库。
- 不要移动 `active_vore/`。
- 不要把 `WEBCLAUDE/` 当源材料目录。
- 不要把机器产物重新散放到公司根目录。
- 不要手工维护 `Agent/.modelking/`。
- 不要把 `Agent/forecast/` 当可长期保存的人工工作区;它会被正式重算清空。
- 不要为凑整洁改动路径而不改代码、skills 和 docs 中的引用。

## 新建或迁移公司的最低检查

1. 根目录有 `公司判断和最新观点.md` 或明确知道它缺失。
2. 根目录只有高频人工工作件,没有散落的 `data.db/defaults.yaml/yaml1*.yaml/forecast/recon/.modelking`。
3. `Agent/` 包含建模运行产物:`data.db/defaults.yaml/yaml1*.yaml/forecast/recon/.modelking` 等。
4. `公告/年报/` 和 `公告/季报/` 存放公告材料。
5. `active_vore/` 保持原位。
6. `WEBCLAUDE/` 只作为打包输出,可删除重建。
7. 路径变更后同步检查 `src/`、`skills/`、`.claude/skills/`、`docs/` 中的路径引用。
