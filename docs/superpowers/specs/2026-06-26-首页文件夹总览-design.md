# 首页 · 文件夹总览 设计

- 日期：2026-06-26
- 范围：workbench 前端 + `src/workbench.py` 后端
- 目标：在本地工作台加一个全局「首页」，其第一个 subtab「文件夹总览」用一张表展示所有公司的文件夹整齐 / 健康程度，并提供「一键归档历史模型」动作。

## 背景

`docs/公司文件夹快速导航.md` 规定了公司文件夹的标准结构与边界，但实际 4 家公司状态差异大且无可视化：

- 新乳业根目录核心假设停在 06024、yaml1 已编译到 06026（日期脱节）；Agent 下 3 个 yaml1 vintage 堆积；根目录 3 个 Model Excel + 锁文件。
- 青岛啤酒只到 data.db/defaults，未建模。
- 会稽山根目录 stale `~$` Excel 锁文件残留。
- 所有公司研报/纪要/收集/重要文件全空。

目前没有一眼看全公司健康度的入口，旧 yaml1 / 根目录 Excel vintage 也没归档机制。

## 非目标

- 不做流水线前置闸门（不在跑 /ka、/comp、forecast 前拦截）。
- 不修改 `公司文件夹快速导航.md` 的标准结构，不新增公司一级目录约定（仅复用 `Agent/` 下 history 目录，镜像 `KAhistory/` 做法）。
- 不碰 `raw_tushare` / `clean_*` / `forecast/`。

## 导航架构

当前 App：`Sidebar(公司列表) + 主区(选中公司的 7 个 tabs)`。

新增**全局「首页」模式**，作为 Sidebar 里一个"特殊的公司"条目，置于 Companies 列表之上，视觉与公司项一致：

- 点「首页」→ `homeActive=true, selectedCompanyId=null`，主区渲染首页视图。
- 首页有自己的 subtab 状态 `HomeTab`，第一项 `folder-overview`，预留扩展位。
- 首页模式下 Sidebar 公司列表仍可见；点任一公司 → 退出首页（`homeActive=false`）进入该公司视图。
- `TabKey` 现有类型不动；首页 subtab 用独立 `HomeTab = "folder-overview"`。

状态：`homeActive` 为 true 时渲染首页，否则渲染现有公司视图。不破坏现有路由与 tab 状态。

## 后端 API

### `GET /api/home/folder-overview`

返回 `Array<CompanyFolderOverview>`，每家公司：

```
{
  company_id, name, code,
  signals: {
    pipeline_stage: "未初始化" | "初始化完毕" | "核心假设完毕" | "建模完毕" | "建模完毕且有DA表",
    yaml1_date: string | null,          // latest yaml1 文件名 YYYYMMDD → "YYYY-MM-DD"
    yaml1_versions: number,             // Agent/yaml1*.yaml 文件数（含 latest）
    yaml1_archive_eligible: boolean,    // yaml1_versions > 1
    root_models: { excel_count, lock_count, archive_eligible },  // 根目录 *.xlsx(排除~$) + ~$*.xlsx
    workbench_materials: { reports, notes, collected, important }, // 研报/纪要/收集/重要文件 file count
    agent_materials: { load, brkd, top_weight, adj, pjbg }        // Skills素材包 5 子目录 file count
  }
}
```

`pipeline_stage` 判定（顺序，命中即停）：

1. 无 `Agent/data.db` → `未初始化`
2. 有 data.db 但无 `Agent/defaults.yaml` → `未初始化`（取数未完成）
3. 无根目录核心假设.md 且无 yaml1 → `初始化完毕`
4. 有核心假设.md、无 yaml1 → `核心假设完毕`
5. 有 yaml1、无 `Agent/da_schedule.yaml` → `建模完毕`
6. 有 yaml1 + `Agent/da_schedule.yaml` → `建模完毕且有DA表`

复用现有 helper：`_company_dirs()`、`_company_dir()`、`_latest_yaml1()`、`_core_assumption()`、`company_defaults_path()`。

### `POST /api/companies/{company_id}/archive-models`

无 body。返回 `{ archived_yaml1: [...], archived_models: [...], deleted_locks: [...] }`。

逻辑：

- **yaml1 归档**：`Agent/` 下所有 `yaml1*.yaml`，按 `latest_yaml1_path()` 判定最新一份留在原地，其余 `git mv` 进 `Agent/yaml1history/`（非 tracked 回退 `shutil.move`），撞名加 `-HHMMSS`。
- **根目录 Excel 归档**：根目录 `*.xlsx`（排除 `~$`），按文件名日期串（`Model-YYMMDD`）取最新 vintage 留原地，其余移进 `Agent/Modelhistory/`；无日期串的视为最旧。
- **锁文件清理**：根目录所有 `~$*.xlsx` 直接删除，列进 `deleted_locks`。
- **守卫**：目标 history 目录不存在则建；`git mv` 失败回退 `shutil.move`；**绝不移动/删除最新 yaml1、绝不删 `~$` 以外文件、绝不碰 `Agent/forecast/`、`recon/`、`.modelking/`**。任一步异常 → 409 + 原因，已成功步骤不回滚（移动是幂等的，重试安全）。

## 前端

- `Sidebar` 顶部加「首页」按钮，className 复用 `company-item` 视觉，selected 态绑定 `homeActive`。
- 首页主区组件 `FolderOverview`：fetch `/api/home/folder-overview`，渲染一张表，列为公司（公司名+代码），行为 6 个信号；或列=信号、行=公司（按导航文档对比表习惯，行=信号、列=公司）。**采用行=信号、列=公司**，与用户提供的对比表一致。
- 6 行：建模管线推进 / yaml1 日期 / yaml1 历史版本 / 根目录模型文件 / 工作台素材数 / Agent技能素材数。
- `yaml1 历史版本` 行：`yaml1_versions > 1` 时显示数字 + 黄色「归档」按钮，点击调 `archive-models`，成功后刷新该列。
- `根目录模型文件` 行：`archive_eligible`（excel_count>1 或 lock_count>0）时显示数字 + 「归档」按钮。
- 归档按钮点击 → 确认气泡（列出将归档的文件数）→ 调用 → toast 结果 → 刷新整表。
- 空公司目录或读取失败：该列 signals 缺失项显示 `-`，不崩表。

## 错误处理

- `/api/home/folder-overview`：单家公司异常不拖垮整表，捕获后该公司返回 `signals=null` + `error` 字段，前端该列显示「读取失败」。
- `/api/companies/{id}/archive-models`：守卫失败（如 latest 判定不了、目标目录无法创建）→ 409，不改文件。
- 文件名日期串解析失败 → 该文件视为最旧，保守归档而非报错。

## 测试 `tests/test_workbench_home.py`

用 `tmp_path` 造公司目录，断言：

1. `pipeline_stage` 5 态：各造一个 tmp 公司目录命中每一态。
2. yaml1 多版本归档：造 3 个 yaml1（不同日期），归档后 `Agent/` 只剩 latest，其余在 `Agent/yaml1history/`。
3. 根目录 Excel：造 2 个 `Model-YYMMDD.xlsx` + 1 个 `~$xxx.xlsx`，归档后根目录只剩最新 vintage，旧 vintage 在 `Agent/Modelhistory/`，锁文件被删。
4. 守卫：latest yaml1 不被动；`~$` 以外文件不被删；`Agent/forecast/` 不被动。
5. `/api/home/folder-overview` 字段完整性：workbench_materials / agent_materials 计数正确。
