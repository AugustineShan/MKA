# 前端使用教程页 — 设计文档

> 日期: 2026-06-23
> 状态: 设计已与用户对齐，待实现

## 1. 背景与目标

MKA 本地工作台（`app/`，React + Vite）目前 5 个顶级 tab（Overview / 核心假设展示 / 完整三表 / DCF / Materials）都是 **per-company** 数据视图。但建模靠一套 CLI/slash-command skills（init / brkd / ka / comp / annual-update / webka / webcomp），新用户无从得知怎么用。

本设计新增一个**全局使用教程页**，教用户怎么用这些 skills：管线全景 + 新手路线 + 每个 skill 一张卡（何时用/命令/输入/输出/下一步/纪律要点）。

## 2. 入口与机制

- **头部加「使用教程」按钮**：放在公司选择器旁，summary 页（公司列表）和 detail 页（公司 tabs）都可见。
- 点击 → 打开**全屏覆盖页**（`position: fixed; inset: 0; z-index` 最高，白底），覆盖当前内容。
- **不绑公司、不占 tab**——符合 `docs/前端设计规范.md` §2"固定五个顶级 tab、顺序不可调"且 tab 是公司数据视图的约束。教程是全局的，不属于任何公司。
- 关闭：右上角 ✕ 按钮 或 Esc 键 → 回到原页面。
- 实现：React state `showTutorial: boolean`，不引入 react-router（与现有 `page` state 架构一致）。`<Tutorial />` 条件渲染在 App 根。

## 3. 内容结构（单页可滚动）

### 3.1 管线全景图（页顶）
主流程横向流程图：
```
研报/纪要 → /brkd → Agent业务讨论.md → /ka → 核心假设.md → /comp → yaml1 → DCF
```
支线三条（主流程下方简注）：
- `init`：刷数据到 clean_annual（全链路前置）
- `annual-update`：年度滚续（出年报后把旧核心假设滚到最新）
- `webka` / `webcomp`：网页端打包（材料搬到 Claude.ai 跑）

用 CSS 变量画（`--blue` 蓝箭头、SF Mono 节点文字），**不引入图表库**。

### 3.2 新手 4 步路线（快速上手）
1. `/init 公司` — 刷数据
2. `/brkd 公司` — 业务预理解（读研报 → Agent业务讨论.md）
3. `/ka 公司` — 核心假设生成
4. `/comp 公司` — 编译 yaml1 + 出 DCF

每步一句话。

### 3.3 每个 skill 一张卡
按顺序：init → brkd → ka → comp → annual-update → webka → webcomp。

卡片固定字段：
| 字段 | 内容 |
|---|---|
| 何时用 | 触发场景（"init 美的集团"、"出年报了更新一下"） |
| 命令 | `/init 美的集团` 等，SF Mono 蓝色 |
| 输入 | 要准备的材料（公司判断.md、研报、defaults.yaml…） |
| 输出 | 产物文件 + 位置（Agent业务讨论.md、核心假设.md、forecast/…） |
| 下一步 | 接哪个 skill |
| 纪律要点 | 1-2 条核心红线（brkd"碰旋钮不拍板"、init"exit 3 不改判"、ka"只认根目录"等） |

## 4. 数据组织

- 内容抽成 `app/src/tutorialContent.ts`：
  - `skills: SkillCard[]` 数组（每项含上述 6 字段 + skill 名 + 管线位置标签）
  - 管线图节点/箭头常量、新手路线常量
- `Tutorial.tsx` 导入该数据，卡片渲染器遍历 `skills[]`。
- 理由：文案集中一处易改，不污染 `App.tsx`（已 3370 行）；后续 skill 增删只改数据文件。

## 5. 风格

遵循 `docs/前端设计规范.md`（Apple HIG）：
- 白与 `#F5F5F7` 灰底、`#1D1D1F` 主文字、交互蓝用 CSS 变量 `--blue`（`#003d7a`）。
- SF Pro 正文；命令/代码用 SF Mono。
- 无渐变、无装饰图标；卡片用浅灰描边或极浅底分隔。
- 卡片网格：桌面 2 列、窄屏 1 列（复用现有响应式断点）。
- 数字/命令右对齐用 SF Mono；与现有表格风格一致。

## 6. 改动文件

| 文件 | 动作 | 责任 |
|---|---|---|
| `app/src/tutorialContent.ts` | 新建 | skills 数据 + 管线/路线常量 |
| `app/src/Tutorial.tsx` | 新建 | 全屏页组件 + 卡片渲染器 + 管线图 + Esc 关闭 |
| `app/src/App.tsx` | 改 | 头部加「使用教程」按钮 + `showTutorial` state + 挂载 `<Tutorial />` |
| `app/src/styles.css` | 改 | 教程页样式（覆盖层、卡片网格、管线图、响应式） |

不动后端、不动 5 个 tab、不引入新依赖。

## 7. 约束与边界

- 教程内容是**静态教育内容**，硬编码在 `tutorialContent.ts`（不从后端拉）。skill 信息稳定，无需动态化。
- 不做搜索/过滤（YAGNI，7 张卡滚动即可）。
- 不做管线节点点击跳转（YAGNI）。
- 教程页不读公司数据，与 `companies/` 无耦合。

## 8. 验收

- summary/detail 两页头部都有「使用教程」按钮。
- 点击 → 全屏教程页，含管线图 + 新手 4 步 + 7 张 skill 卡。
- ✕ / Esc 关闭回原页。
- `npm run build` 通过（TS 类型检查 + 构建）。
- 窄屏（< 768px）卡片 1 列、管线图可横滚或换行。
- 风格符合设计规范（无渐变、SF Pro/Mono、CSS 变量色）。
