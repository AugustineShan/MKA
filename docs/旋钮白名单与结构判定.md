# 旋钮白名单与结构判定（单一参考）

> 本文是 audit 产物：把散在 `核心假设调整器_skill`、`frontend-edit` SKILL、`adj` SKILL、`年度更新器_skill` 里各自重述的"quick 可拨什么 / 禁止什么 / 什么算结构改动"收成一处单一参考。
> `/adj quick`、`/frontend-edit`、`/adj incremental`、`/annual-update` 凡涉及旋钮边界或结构判定，**指向本文**，不要再各自重抄，以免改一处漏几处。
> 改这三类约定时改这里 + 引用方，别平行重述。

## 一、quick 可拨旋钮白名单（不可加宽）

quick 与 frontend-edit 定点 patch 只允许改以下已存在条目的 `values[i]`（或标量的标量值）。新增任何 path、结构、slug、family、horizon 或 fade 编辑都不属 quick，转 `/adj incremental` 或重走 `/comp`。

| 旋钮 | yaml1 path | unit | 说明 |
|---|---|---|---|
| 整体毛利率 | `income.gpm.values[i]` | pct | |
| 成本费用率 | `income.cost_rates.*.values[i]` | pct | 销售费用→sell_exp / 管理费用→admin_exp / 研发费用→rd_exp / 营业税金及附加→biz_tax_surchg |
| 有效税率 | `income.effective_tax_rate.values[i]` | pct | |
| 少数股东损益比 | `income.minority_ratio.values[i]` | pct | |
| 其他财务费用绝对值 | `income.financial_expense.other_fin_exp_abs.values[i]` | abs_mn | financial_expense 族；frontend-edit 须补登记映射行（见下） |
| 收入 leaf（因子族） | 已存在 `projection.values[i]` | pct | factor_product/vol_price 族，因子级 |
| 收入 leaf（growth/abs 族） | 已存在 `knobs.revenue_yoy[i]`/`revenue_abs[i]` | pct/abs_mn | segment 级 |
| below-OP 绝对值 | 已存在 `income.operating_adjustments_abs.{field}.values[i]` 或 `income.below_line_abs.{field}.values[i]` | abs_mn | 以实际 yaml1 为准 |
| 永续增长 | `terminal.perpetual_growth` | scalar | 标量，非 values 数组 |

> `frontend-edit` 的 IS 旋钮路径映射表须包含 `financial_expense` 族（与上表一致）。前端新增旋钮族须先在 `frontend-edit` SKILL 映射表登记，并在本表登记；**未登记的 path → 报错停止并提示登记，不静默 patch**。

## 二、quick 禁止清单（单一超集）

下列编辑一律不属 quick，须转 `/adj incremental`（开骨架门）或视情况弹回 `/ka`。

| 业务语言 | yaml1 path / 结构 |
|---|---|
| 改显式期长度 | `terminal.explicit_end` / `meta.horizon` |
| 改衰减期长度 | `terminal.fade.to_year` |
| 改衰减交接增速 | `terminal.fade.target_growth` |
| 新增旋钮 path | 新增 path |
| 删除旋钮 path | 删除 path |
| 改参数化族 | 改 `kind` / family（如 growth↔factor_yoy、费用率↔绝对值） |
| 改业务线 slug | 改 segment slug |
| 整体毛利与分线毛利互相翻转 | gpm 族 ↔ 分线毛利率 |
| 改历史事实 | 历史区 |
| 改收纳区 | stash |
| 改来源说明 | 来源 |
| 发明 yaml1 path | yaml1 结构变更 |

## 三、结构改动判定表（可观测特征 → 处理路径）

替代"推翻裁决基础 / 参数化翻转 / 结构性变化 / 改参数化"等主观同义异名。判据看可观测特征，不看意图。

| 变化特征 | 处理路径 | 开骨架门？ |
|---|---|---|
| 只改已有 knob 的 `values[i]`（或 `terminal.perpetual_growth` 标量） | `/adj quick` 或 `/frontend-edit` | 否 |
| 新增/删除收入线或费用线 | `/adj incremental` | 是 |
| 改参数化族（growth↔factor_yoy、费用率↔绝对值等） | `/adj incremental` | 是 |
| 改 horizon / fade 长度 / `terminal.fade.target_growth` | `/adj incremental` | 是 |
| thesis 方向反转（如扩张转收缩、增长假设整体推翻） | 弹回 `/ka` 重建 | — |
| 年度滚轴内的时间迁移（旧稿 H→A） | `/annual-update` | 按年度更新器纪律 |

> "thesis 方向反转"是唯一弹回 `/ka` 的条件；其余结构改动留在 `/adj incremental` 开骨架门处理。判不定时偏保守：留在 incremental，不擅自降级 quick、也不擅自升级 /ka。
